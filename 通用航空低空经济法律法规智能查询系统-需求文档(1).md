# 通用航空与低空经济领域法律法规文件智能查询系统需求文档

## 1. 项目概述

### 1.1 项目背景
通用航空与低空经济领域法律法规文件数量庞大、更新频繁，传统的查询方式效率低下。本项目旨在构建一个智能化的法律法规文件查询系统，为用户提供高效、便捷的法规检索、订阅和管理服务。

### 1.2 项目目标
- 构建专业的通用航空与低空经济领域法律法规知识库
- 提供智能化的全文检索和场景化搜索能力
- 实现用户分级权限管理和数据隔离
- 支持法规更新及时推送和订阅提醒
- 满足等保测评要求，确保系统安全可靠

### 1.3 应用场景
- 通用航空企业查询合规要求
- 监管部门执行法规审核
- 行业从业者获取最新法规动态
- 系统集成方调用API获取法规数据

## 2. 技术架构

### 2.1 技术栈

#### 2.1.1 后端技术
- **开发框架**: Django 4.x
- **编程语言**: Python 3.10+
- **数据库**: PostgreSQL 14+
- **全文搜索引擎**: Elasticsearch 8.x
- **向量数据库**: 用于RAG检索
- **缓存**: Redis 6+
- **消息队列**: Celery + RabbitMQ

#### 2.1.2 前端技术
- **开发框架**: React 18
- **UI组件库**: 待定（TDesign/Ant Design等）
- **状态管理**: Redux Toolkit
- **HTTP客户端**: Axios
- **构建工具**: Vite

#### 2.1.3 部署架构
- **容器化**: Docker + Docker Compose
- **Web服务器**: Nginx
- **应用服务器**: Gunicorn
- **部署环境**: 云服务器
- **架构模式**: 单节点部署（不需要集群）

#### 2.1.4 第三方服务
- **大语言模型**: 用于RAG法规通俗解读
- **文件存储**: MinIO/OSS
- **短信服务**: 用于双因素认证
- **邮件服务**: 用于系统通知

## 3. 功能需求

### 3.1 用户认证模块

#### 3.1.1 用户注册与登录
- **登录方式**
  - 账号密码登录
  - 手机号登录（支持短信验证码）
- **用户类型**
  - 个人用户
  - 企业用户（需提供企业信息）
- **双因素认证（2FA）**
  - 普通用户：不需要
  - 监管用户：不需要
  - 管理员：必须启用
- **会话管理**
  - JWT Token认证
  - Token过期时间可配置
  - 支持刷新Token机制

#### 3.1.2 用户管理
- 用户信息维护（头像、昵称、联系方式等）
- 企业用户资质管理（营业执照、资质证书等）
- 密码修改与找回
- 账号注销

### 3.2 知识库管理模块

#### 3.2.1 法规文件管理
- **文件上传**
  - 支持PDF格式
  - 批量上传功能
  - 文件大小限制（单文件≤50MB）
- **文件解析**
  - 自动提取文本内容
  - 提取元数据（标题、发布日期、发文部门、文号等）
- **文件分类**
  - 四大场景分类：航空器管理、飞行安全、运行监管、空域使用
  - 支持多级分类目录（可扩展至5级）
- **文件维护**
  - 文件编辑（元数据修改）
  - 文件版本管理
  - 文件下架与删除

#### 3.2.2 数据采集
- **爬虫采集**
  - 定期爬取官方发布渠道
  - 支持多个数据源配置
  - 去重机制避免重复采集
- **API监听**
  - 监听官方API数据推送
  - 实时接收新法规文件
- **数据处理**
  - 数据清洗与标准化
  - 文件格式转换（如需要）
  - 索引更新触发

#### 3.2.3 知识库统计
- 法规文件总数统计
- 各分类文件数量统计
- 文件更新频率统计
- 文件访问热度排行

### 3.3 检索引擎模块

#### 3.3.1 搜索功能
- **搜索方式**
  - 关键词全文搜索
  - 标题精确搜索
  - 场景化意图解析（根据搜索意图推荐相关法规）
- **搜索范围**
  - 可按分类筛选
  - 可按发布部门筛选
  - 可按时间范围筛选
  - 可按文件类型筛选
- **搜索结果**
  - 关键词高亮显示
  - 搜索结果摘要展示（提取相关段落）
  - 单次搜索结果无上限，支持分页展示
  - 默认每页20条结果，可调整
- **搜索排序**
  - 相关度排序（默认）
  - 发布时间排序（最新/最早）
  - 访问量排序

#### 3.3.2 搜索历史
- 记录用户搜索历史
- 支持清除搜索历史
- 支持从历史记录快速搜索
- 搜索历史保留90天

#### 3.3.3 热门搜索
- 全局热门搜索词排行
- 实时更新热门搜索统计
- 首页展示热门搜索推荐

#### 3.3.4 搜索性能
- 搜索响应时间：毫秒级（目标<500ms）
- 支持100并发用户
- 搜索结果缓存机制

### 3.4 订阅推送模块

#### 3.4.1 订阅管理
- **订阅类型**
  - 按时效订阅：新法规发布通知
  - 按部门订阅：特定部门法规发布通知
  - 按分类订阅：特定分类法规发布通知
  - 按关键词订阅：包含特定关键词的法规通知
- **订阅操作**
  - 添加订阅
  - 修改订阅
  - 删除订阅
  - 查看订阅列表

#### 3.4.2 消息推送
- **推送方式**
  - 站内消息推送（默认）
  - 即时推送
- **推送内容**
  - 法规文件标题
  - 发布日期
  - 简要内容摘要
  - 快速跳转链接
- **消息状态**
  - 已读/未读标记
  - 历史提醒记录保留（永久）
  - 支持全部标记为已读

#### 3.4.3 登录弹窗提醒
- **触发条件**
  - 用户登录后检测到有未读的订阅提醒
  - 法规更新后首次登录
- **弹窗内容**
  - 法规更新数量
  - 法规列表（标题+发布日期）
  - 查看详情按钮
  - 暂不查看按钮
- **交互逻辑**
  - 用户确认后弹窗消失
  - 未读消息数更新
  - 消息状态标记为已推送

### 3.5 权限管理模块

#### 3.5.1 用户角色
- **普通用户**
  - 搜索法规文件
  - 查看法规详情
  - 订阅法规更新
  - 下载法规文件
  - 查看统计报表（个人维度）
- **监管用户**
  - 继承普通用户所有权限
  - 管理审核内容
  - 查看全量法规文件
  - 查看用户搜索记录（脱敏）
  - 导出审核报表
- **管理员**
  - 继承监管用户所有权限
  - 管理用户（增删改查、角色分配）
  - 维护知识库（上传、编辑、删除文件）
  - 系统配置管理
  - 查看系统日志
  - 数据备份与恢复

#### 3.5.2 权限控制
- **RBAC权限模型**
  - 基于角色的访问控制
  - 支持自定义角色和权限
- **数据权限隔离**
  - 普通用户：只能查看公开法规
  - 监管用户：查看公开+待审核法规
  - 管理员：查看所有法规
- **API权限**
  - API调用需认证
  - 不同角色API访问权限不同

### 3.6 统计分析模块

#### 3.6.1 用户统计
- 用户注册数量趋势
- 用户活跃度统计
- 用户类型分布
- 用户登录频次统计

#### 3.6.2 搜索统计
- 搜索次数统计（日/周/月）
- 热门搜索词排行
- 搜索无结果率统计
- 搜索响应时间分布

#### 3.6.3 法规统计
- 法规文件数量趋势
- 各分类法规数量分布
- 法规访问热度排行
- 法规下载次数统计

#### 3.6.4 报表导出
- 支持Excel格式导出
- 支持PDF格式导出
- 支持自定义报表时间范围
- 管理员可导出全量报表

### 3.7 系统管理模块

#### 3.7.1 数据管理
- **数据备份**
  - 定期自动备份（每日增量，每周全量）
  - 手动触发备份
  - 备份文件保留30天
- **数据恢复**
  - 支持从备份恢复
  - 恢复前需确认

#### 3.7.2 日志管理
- **操作日志**
  - 记录用户所有操作
  - 记录系统关键操作
  - 日志保留90天
- **敏感数据脱敏**
  - 手机号脱敏（中间4位隐藏）
  - 身份证号脱敏
  - 邮箱脱敏
- **日志查询**
  - 按用户查询
  - 按操作类型查询
  - 按时间范围查询

#### 3.7.3 系统配置
- 系统参数配置（搜索分页大小、缓存时间等）
- 数据源配置（爬虫源、API接口）
- 消息推送配置（推送频率、推送模板）
- 安全策略配置（密码强度、Token过期时间）

## 5. 数据库设计

### 5.1 核心数据表

#### 5.1.1 用户表 (users)
```sql
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    phone VARCHAR(20),
    email VARCHAR(100),
    user_type VARCHAR(20) NOT NULL, -- individual, enterprise
    role VARCHAR(20) NOT NULL,      -- user, regulator, admin
    enterprise_name VARCHAR(200),
    business_license VARCHAR(100),
    avatar VARCHAR(500),
    is_active BOOLEAN DEFAULT TRUE,
    is_2fa_enabled BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 5.1.2 法规文件表 (laws)
```sql
CREATE TABLE laws (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(500) NOT NULL,
    content TEXT,
    category_id BIGINT,
    department VARCHAR(200),
    document_number VARCHAR(100),
    publish_date DATE,
    effective_date DATE,
    file_url VARCHAR(500),
    file_size BIGINT,
    view_count BIGINT DEFAULT 0,
    download_count BIGINT DEFAULT 0,
    status VARCHAR(20) NOT NULL, -- draft, published, archived
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES law_categories(id),
    FOREIGN KEY (created_by) REFERENCES users(id)
);
```

#### 5.1.3 法规分类表 (law_categories)
```sql
CREATE TABLE law_categories (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    parent_id BIGINT,
    level INTEGER NOT NULL,
    sort_order INTEGER DEFAULT 0,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES law_categories(id)
);
```

#### 5.1.4 订阅表 (subscriptions)
```sql
CREATE TABLE subscriptions (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    subscription_type VARCHAR(50) NOT NULL, --时效/部门/分类/关键词
    target_value VARCHAR(200) NOT NULL, -- 订阅的具体值
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 5.1.5 消息通知表 (notifications)
```sql
CREATE TABLE notifications (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    law_id BIGINT,
    title VARCHAR(500) NOT NULL,
    content TEXT,
    is_read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (law_id) REFERENCES laws(id)
);
```

#### 5.1.6 搜索历史表 (search_history)
```sql
CREATE TABLE search_history (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT,
    keyword VARCHAR(500) NOT NULL,
    search_type VARCHAR(20), -- fulltext, title, intent
    result_count INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 5.1.7 操作日志表 (operation_logs)
```sql
CREATE TABLE operation_logs (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT,
    action VARCHAR(100) NOT NULL,
    resource_type VARCHAR(50),
    resource_id BIGINT,
    ip_address VARCHAR(50),
    user_agent TEXT,
    request_data TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### 5.1.8 系统配置表 (system_config)
```sql
CREATE TABLE system_config (
    id BIGSERIAL PRIMARY KEY,
    config_key VARCHAR(100) UNIQUE NOT NULL,
    config_value TEXT,
    description TEXT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|

| 爬虫数据采集失败 | 中 | 中 | 多数据源冗余，设置采集监控和告警 |
| 等保测评不通过 | 高 | 低 | 提前按照等保要求开发，邀请第三方安全评估 |

| 风险 | 影响 | 概率 | 应对措施 |
